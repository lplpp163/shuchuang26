# 傳家話｜說一句・演成我們家的故事

這是一個 Flutter 跨平台競賽原型。第一次開啟可先進行約 30 秒三幕試演：選一句、看舞台改變，再直接看懂「孩子帶回—家人傳下—孩子接住」；固定合成、零家庭資料。正式使用時，孩子不是進入題庫，而是和家庭角色演一段 60–90 秒的生活故事：孩子自己點擊播放長輩首句，再用麥克風或圖片短詞接話；角色回應後，故事與場景隨選擇改變，最後留下可由家人接續的私密故事卡。

> 別人教「標準答案」；這個作品保存「我們家真的會怎麼說」。

第一個示範語言是越南語。內建句仍是合成語音示範，尚未完成母語者逐句審閱；家庭版本永遠由真人家人決定，系統不替地方口音排名。

## 孩子的核心體驗

首頁只有一個主要任務「今天的家庭劇場」，不再先顯示連勝、XP、四種模式或長篇教材。

每集固定三回合，每回合至少兩條都能推進的故事分支：

1. 每幕先顯示家庭角色、目標語、情境插圖與中文意思；長輩首句不會自行播放，孩子點擊播放鍵後才發聲。
2. 孩子先選「想表達的意思」，看見短短的目標語、羅馬拼音與中文；可正常重聽、慢速再聽，或逐段點擊 `/` 分段後再按「換我說」。逐段固定使用裝置朗讀，不假裝切割家人原音。
3. 若裝置聽不清楚、沒有權限或不支援該語言，角色自然請孩子再說一次，並立即提供圖片與短詞卡；故事不會中斷。
4. 孩子送出選擇後，角色回一句，舞台保留同一對祖孫角色並改變色彩、路徑與物件；回話結束後可倒數接到下一幕，孩子也能「先停一下」或馬上接。這個換幕倒數不代表下一幕首句會自行發聲；減少動畫／輔助瀏覽模式永遠改用手動。
5. 前兩次選擇會一路保留到第三幕，不在中段偷偷合流；下一問也延續剛才改變的世界，不跳回開場照片。
6. 第三回合只慶祝一次，建立包含完整選擇路徑的家庭故事卡。
7. 真人家人可稍後加貼圖、錄下一句或留一張家庭便條；孩子當下完全不用等待。
8. 每張卡會留一個具體的家庭記憶問題，例如「哪一道菜一端上桌，你就知道這是我們家？」；文化版本交回真人家人回答。

目前內建五個可完整遊玩的生活劇場：

- 放學回家
- 早安！起床囉
- 一起準備晚餐
- 陽台澆花
- 睡前故事

所有劇場都能在沒有遠端生成服務時完成；情節是可審核的本機狀態機，不會因模型漂移產生不適合兒童的內容。

## 五個家庭接力題材與三筆教育機會情報

「選故事」頁首已顯示五個可點選的家庭接力題材：和家人分享、社團、午餐、上課、朋友關係。孩子先從兩個生活意圖選出自己想說的「第一棒」，再把裝置交給家人；家人通過 PIN 後確認真正說法並可錄家人原音。孩子接著走完既有的「看情境 → 聽短句 → 重排 → 回答」四關，完成時揭曉「孩子帶來 → 家人傳下 → 孩子接住」三棒作品，並可在家人圈重播成人與孩子留下的本機錄音。

「家庭故事護照」只依已完成的接力蓋章：只選題、完成成人 PIN 或存下家人短句都不算完成。這五項是建立在既有四關任務上的家庭接力，不是五個新的三回合劇集，也沒有被算入 119 個內建 Piper 示範音檔。原有完整三幕劇場仍是放學回家、早晨起床、一起準備晚餐、陽台澆花與睡前故事五集。

App 同頁已實作「3 筆官方教材・課程・競賽入口」，內容於 2026-07-14 查核下列官方來源：

- [115年全國性多語多元文化繪本親子共讀心得感想甄選](https://mkm.k12ea.gov.tw/news/17202605110001)：2026-09-21～2026-10-23 收件，由學校統一報名。App 依日期顯示「即將收件」、「收件中」或「已截止」。
- [新住民子女教育資訊網](https://mkm.k12ea.gov.tw/)：持續更新教材、課程、研習與競賽公告。
- [115年全國新住民語文說故事競賽成果](https://www.moe.gov.tw/News_Content.aspx?n=9E7AC85F1954DDA8&s=9C06AFB083644471&sms=169B8E91BB75571F)：活動已於 2026-05-30 辦理，只標示為「已結束・成果參考」，不冒充仍可報名。

每筆情報都顯示官方主辦、查核日與「以官網為準」，並提供一個不離開 App 的本機主題延伸動作，讓家庭把公開主題接到既有故事或家庭接力。這些延伸內容逐筆明示「主題延伸，非官方授權教案」；使用者必須另外明確按下「和家人開啟官方頁」才會外連。平台無法開啟時，App 會保留在原畫面並顯示錯誤提示。所有期程與報名資格仍以官網最新公告為準；這三筆入口與本機延伸都不代表本作與教育部、國教署、學校或活動主辦單位已有合作或獲得推薦。

另已產出 19 頁 A4 的「傳家話教師家庭延伸情境包｜四週試辦版」，提供四週流程、家庭任務、觀察紀錄與官方主題對照。它是可執行但尚待真人試辦的導入工具，不是官方教材、認證教案、已採用方案、成效證明、合作或採購文件；其中價格只作為待訪談假設，不是報價或成交證據。

## 語音能力的誠實邊界

目前有三種彼此分開的能力：

- `speech_to_text`：把孩子主動說出的短句轉成文字；系統只在裝置上尋找已知選項的關鍵詞，模糊或多重符合時會請孩子再選一次。
- `LocalPracticeCoach`：只看錄音秒數與麥克風音量，提示節奏與收音。
- 家人親耳確認：決定意思是否聽懂、家裡平常怎麼說。

這些都不是音素、聲調、地方腔或正式發音評量。低信心的聽寫即使文字碰巧吻合，也不會自動判定成功；畫面不顯示紅叉、百分比或數字分數。

播放層也採取同一個誠實原則：

- 約 30 秒試演與正式劇場的每幕首句都要由使用者明確點擊播放；進入頁面本身不作為播放授權，也不拿舊版頁面載入即發聲的結果當成目前證據。
- 生活場景中央的大耳朵一律播放完整 `story.audioPath`，不再依孩子點到的答案偷換成末段音檔；修正前最短甚至可能只播單獨的 `ạ`。
- 教材不再把末尾單詞硬拆成缺少語境的孤立音檔；會併回前一段，完整句、分段與替換例句各自綁定可檢查的播放來源。
- `DeviceSpeechService` 先正規化 BCP-47 語言標籤，再等待並重試精確語言的裝置 voice；找不到相符 voice 或語言未知時直接停止並顯示可理解的 fallback，不交給中英文預設 voice 猜讀。
- 目前實際引用的 119 個離線 MP3 全部由同一個釘選 Piper 版本與越南語模型離線產生；App 執行時不依賴 Piper，也不混用來源不明的候選音檔。家庭自訂但未錄音的文字才使用裝置 TTS。
- `assets/audio/piper_generation_manifest.json` 記錄產生器、模型、模型 SHA-256、授權、每個檔案的輸入文字、bytes 與 SHA-256；`tool/bundled_audio_catalog.dart` 保留 119 條實際引用路徑與文字。
- Web 與原生平台使用分開校正的正常／慢速 preset；播放請求採 latest-wins，停止或新請求會使舊請求失效，避免連點後舊聲音覆蓋新聲音。
- `FamilyStory.languageTag` 會隨故事持久化；未知語言保存為 `und`，不再一律猜成中文或越南語。

119 個隨 App 打包的檔案是一套可重現的 Piper 合成操作示範，不是真人家人原音。檔案數、hash、文字與路徑綁定都不能證明聲調、語氣或家庭稱謂已達母語正確；正式展示前仍須由越南語母語者或家人逐檔複聽確認。

## 私密家庭圈

家庭圈目前是本機優先、具本人接受儀式的可操作雛形：

- 成員包含關係稱謂、孩子看到的暱稱、角色圖示與成人／孩子權限。
- 不會預置或假造家人；第一位成人要親自輸入長輩與孩子稱呼並確認建立家庭圈。
- 管理者建立一次性邀請包後，受邀成人要親自設定自己的六位數家人碼並交回簽署回覆；原家庭驗證成功後才正式核准，舊的直接核准成人路徑已在資料層封鎖。
- 受邀成人可只用自己的家人碼進入回應故事，不需知道四位管理碼；他看不到邀請、移除成員、備份、匯入或刪除等管理工具。
- 過期、撤銷、重播、內容竄改、錯家庭與嘗試用備份更換管理者都會拒絕，不新增任何角色。
- 家庭管理者可以撤銷待用邀請或移除後來加入的角色；該角色的貼圖與留言會一起移除。
- 每位家人每張故事卡只能留一個限定貼圖，可更換或撤回。
- 只有核准成人能留一句家庭便條，並可選擇錄下 12 秒內的家人原音；這句話會讓孩子下次打開時看到，不會假裝自動生成新劇集。
- 成人可展開五集的三個回合，替每個長輩對話節點留下「我們家的說法」；劇場優先播放對應家人原音，沒有錄音才清楚標示並使用裝置朗讀。
- 新回應會讓其他家人看到未讀提示；孩子打開家人圈後才標為已讀。
- 家庭圈首屏整理最新故事、貼圖、文字／錄音回覆與未讀，讓孩子先看見家人剛留下什麼，再進故事卡細節。
- 「一起完成家庭故事章」只從實際故事卡、主劇情家人原音與成人留言計算 0–3/3；沒有倒數、連勝、排行榜或失敗懲罰，也不阻塞孩子。
- 沒有陌生人搜尋、公開動態、自由私訊、追蹤數或排行榜。
- 管理者可複製／帶入完整文字備份；不會自動同步，錄音檔本體也不會塞進剪貼簿。備份不含個別家人碼 verifier、邀請私鑰或重試狀態。
- 文字、成員、故事卡、回應與讀取狀態都能 JSON 匯出、驗證後匯入或刪除。
- 管理碼與個別家人碼都使用 PBKDF2-HMAC-SHA256 600,000 次的加鹽 verifier；每位家人連續五次輸錯會各自暫停 30 秒，重開後仍保留。這仍是本機原型，不是伺服器帳號或安全硬體。

正式跨裝置即時同步尚未接入。邀請與接受回覆可用可信方式手動交換，但介面會直接標示它不是雲端送達；故事、錄音、帳號復原與推播都不會假裝已同步。

## 家人用幾行話建立內容

`QuickChallengeDraftGenerator` 是供應商中立介面；目前使用 `LocalQuickChallengeDraftGenerator`：

- 五個家庭接力題材各有可檢查的本機生活模板，會把孩子選的第一棒、家人確認的短句與既有固定插圖四關任務綁在一起。
- 未知句子不會假裝翻譯，必須由家人填入並確認真正說法。
- 只有能對上既有固定插圖與題目的模板才會開啟「看、聽、排、答」四關；不支援的自由場景不會硬配一張無關圖片或冒充完成任務。
- 圖片都是預先打包的作品資產；執行時不會假裝即時生圖。
- 家人可選擇錄原音；孩子完成四關後才會完成三棒接力、建立可重播的家人圈作品並替該題材護照蓋章。

未來接入其他後端時，可以替換草稿產生器或對話意圖服務，不需要把供應商憑證放進 Flutter Web。

## 三分鐘 Demo

1. 在同意頁按「先試演約 30 秒」，確認畫面標示不錄音、不儲存，也沒有建立家庭資料。
2. 在試演舞台按「點一下聽這一句」，再選兩個意思之一；確認進入 2／3 舞台結果後，點「看這句怎麼傳回家」進入 3／3，核對三棒、Piper 合成／非真人原音與零家庭資料揭露，最後按「同意後建立我們家的三棒故事」返回同意頁。
3. 回到同意頁，勾選兩項同意、設定四位數家長碼，再由成人親自建立家庭圈。
4. 首頁按「進入放學回家」；先明確點擊播放長輩首句，再選想表達的中文意思，跟著短句與羅馬拼音準備後按「換我說」。沒有可用麥克風時可用短詞卡繼續。
5. 選「我回來了」後確認門打開、外婆回話與場景動畫，再走完另外兩回合。
6. 完成後查看三句選擇組成的家庭故事卡，進入「家人圈」確認孩子不用等待就已完成故事。
7. 右上角「交給家人」先展示孩子／成人交接，再由家庭管理者回應一個限定貼圖、留一句或錄一段家人原音。
8. 展示完整邀請握手：管理者做邀請包、受邀者設定自己的六位碼並做回覆包、原家庭核准後，受邀者只用自己的碼進入回應。
9. 回孩子端查看未讀提示與家人回覆；待核准角色不會出現在孩子成員列。
10. 進入「選故事」，確認頁首可直接看到和家人分享、社團、午餐、上課與朋友關係五個家庭接力題材、0／5 故事護照，以及 3 筆官方資訊入口。
11. 點「社團」等題材，由孩子先選第一棒；把裝置交給家人後以 PIN 進入成人工具，確認真正說法並可錄原音，再交回孩子。
12. 讓孩子走完「看情境 → 聽短句 → 重排 → 回答」四關，確認完成後揭曉三棒接力作品、護照才由 0／5 變成 1／5，並可在家人圈重播。
13. 打開任一教育資訊，分別展示本機主題延伸與官方外連；確認延伸畫面明示「非官方授權教案」，不把資源入口冒充合作或採用。
