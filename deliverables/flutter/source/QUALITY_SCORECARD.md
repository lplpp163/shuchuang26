# 傳家話｜家庭對話劇場品質量表

本量表用來阻止「功能很多，但孩子沒有真的學會或玩起來」的假進度。每次可交付版本都要以 0–10 分重評；證據必須來自實際畫面、可重現流程或自動化測試，不能只看文案與設計稿。

## 一票否決

出現以下任一項時，整體不得高於 5 分：

- 沒有開口卻顯示「說對了」或「發音正確」。
- 只用錄音秒數、音量或一般聽寫宣稱能評估音素、聲調或家庭腔調。
- 宣稱即時生成圖片、聲音、任意題型，實際只是在三個固定模板中挑選。
- 孩子要等待家人回覆才能完成當下故事。
- 兒童聲音、姓名或照片預設公開，或存在陌生人搜尋、私訊、公開排行。
- 同一趟流程出現兩次「完成」，或主要按鈕點下後沒有可見結果。

## 八個評分面向

| 面向 | 8 分門檻 | 10 分狀態 |
| --- | --- | --- |
| 幼兒可懂 | 每屏一個目標、一個主要動作、文字分層清楚，首次使用不需成人解說 | 不識目標語的孩子也能只靠角色、動作、聲音與漸進提示完成 |
| 故事沉浸 | 角色有動機，孩子回答會改變下一句與畫面，60–90 秒內有開場、轉折、收束 | 多次重玩仍有新結果，角色關係與上次選擇會延續 |
| 語音真實性 | 真的錄音／聽寫，能力與限制在動作前說清楚，低信心時自然修復 | 使用經兒童與家庭口音驗證的模型，能提供單一、可操作且可信的發音提示 |
| 遊戲性 | 語音會讓角色或場景立即發生事情，兩個選擇都能推進而非真假答案 | 語言本身就是操控方式，孩子因好奇故事結果而重玩 |
| 家庭文化 | 家人稱謂、原音與家庭版本是內容核心，AI 不決定唯一正解 | 故事取材自家庭記憶並形成可攜、可保存的跨世代聲音作品 |
| 私密社群 | 成人核准、邀請制、不可搜尋、限定反應、可刪除匯出 | 家庭成員能安全接續故事，真人參與有價值但不阻塞孩子即時體驗 |
| UI 聚焦 | 孩子首頁只有一個主要任務；系統術語與家長工具退出孩子主流程 | 角色、動作與世界狀態取代大部分卡片、說明與儀表板 |
| 技術誠實 | 所有功能名稱符合實作；fallback 可用且不假裝雲端或生成式能力 | 能力偵測、資料流與供應商切換都透明，失敗仍能自然完成故事 |

## 每輪要求

1. 先跑全部單元與 Widget 測試。
2. 以手機寬度走完新手第一集、低信心修復、兩個分支與故事卡。
3. 搜尋禁用／高風險文案：未開口卻宣稱答對、精準發音評估、AI 聽懂度、即時生成及供應商專屬名稱。
4. 記錄八項分數、最弱兩項與下一輪具體修改。
5. 所有項目至少 8 分、且一票否決為零，才可宣稱核心體驗完成。

本輪成立的改進：

- 成人不能再由舊 `inviteMember → approveMember` 路徑直接加入。管理者只保存一次性邀請的 Ed25519 公鑰；32-byte 私密 seed 只出現在交給受邀者的邀請包。
- 受邀者本人設定六位數家人碼並簽出接受回覆；原家庭驗證簽章、家庭 ID、成員 ID、期限與時間後才核准。過期、撤銷、竄改、錯家庭與重播全部拒絕。
- 個別家人碼以 PBKDF2-HMAC-SHA256 600,000 次、獨立 16-byte salt 與 32-byte verifier 保存，固定時間比較；每位成人各自五次／30 秒鎖定，重開後仍保留。同一成員的並行驗證會排隊，不能覆蓋失敗次數。
- 受邀成人可只用自己的六位碼進入回應故事，不需知道四位管理碼，也看不到邀請、移除、備份、匯入或刪除工具。
- `managerMemberId` 已成為持久化授權資料；完整備份不能換家庭 ID 或管理者，加入 self-approved 假管理者也會 fail closed。
- 舊版四位管理碼成功驗證一次後，會由快速 SHA-256 自動遷移到 PBKDF2 v2；錯誤次數與鎖定也改為持久保存。
- portable family backup 明確排除家人碼 verifier、salt、待用／已用邀請與 retry state；回覆包必須先複製才能完成，不再留下看似完成、其實無法交回的斷點。

仍不能宣稱產品級雲端社群。Ed25519 證明的是持有一次性邀請 token，現實本人仍依賴家人用可信方式親自交付；目前沒有伺服器帳號、安全跨裝置同步、帳號復原、推播或 E2EE，portable backup 也不搬個別 credential。這些限制已在介面明示，不能用本機手動交換冒充遠端送達。

### Round 8｜2026-07-13｜完整句播放、裝置語音可靠性與語言標籤

最終驗證：品質閘門通過、`flutter analyze` 0 問題、Flutter 測試 106／106、Web release 建置 82 檔、Edge Playwright 3／3、Pixel 7 視覺稽核 1／1。一票否決 0、P0 0。

本輪修正的是「畫面寫完整句，實際卻可能只播答案末段」的教學完整性缺口：

- 場景中央大耳朵改為固定播放完整 `story.audioPath`；測試逐一走過五個場景，防止再次退回末段音檔或單獨 `ạ`。
- 過短的末尾單詞併回前段；當輪教材子集的完整句、分段與替換例句路徑皆有存在性與播放綁定測試。
- `DeviceSpeechService` 只選精確 BCP-47 語言的 voice，支援延遲 voice 清單重試；無相符 voice、未知語言或設定失敗時 fail closed，不以預設中英文 voice 硬念。
- Web 正常／慢速 preset 已獨立校正；播放併發改為 latest-wins，`cancel` 與新請求都會使舊請求失效，避免舊聲音或舊完成狀態覆蓋目前操作。
- `FamilyStory.languageTag` 納入持久化與舊資料相容；`und` 不再猜成中文，自建內容只有在語言已知或已有錄音時才提供朗讀。

技術邊界不因測試數增加而改變：本輪只能證明當時教材子集存在、可解析、綁定到預期按鈕，並證明裝置 TTS 找不到正確 voice 時會安全停止；不能宣稱越南語聲調、語氣、稱謂或分段已經母語正確。正式送件前仍須由越南語母語者或家人逐句、逐檔複聽並留下修訂紀錄。這是歷史紀錄；目前交付的完整音訊基準以 Round 10 的 119 個釘選 Piper 資產為準。
