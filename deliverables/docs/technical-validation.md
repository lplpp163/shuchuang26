# 傳家話｜技術驗證紀錄

文件更新：2026-07-14

## 目前驗證狀態

- 同意前試演以「選一句／看故事變／傳回家」具名三幕呈現；兩個圖像選擇直接放在場景裡，外婆的回話泡泡與動作留在同圖。四段聲音分別標示外婆／孩子／外婆／家人接力，三棒內容不同且依序亮起。正式建立家庭、啟用麥克風或保存故事卡仍在同意之後。
- 第三幕固定揭露Piper合成、非真人原音、尚非母語審閱成果與零家庭資料。Widget逐項比對退出前後stories、attempts、relays與consent不變；320×640、200%文字可走完三幕，主要互動至少48dp。
- 「傳家話」產品 shell 以手動播放模式開啟正式劇場；每幕長輩首句要由使用者明確點擊，不能引用舊版冷啟即發聲事件作為目前證據。
- `assets/audio/` 目前有 119 個實際引用 MP3；全部由同一套釘選 Piper 產生流程輸出，來源、版本、模型 hash、授權與逐檔 hash 記在 `assets/audio/README.md` 及 `piper_generation_manifest.json`。
- 目前可採信的Flutter gate：格式70檔無漂移、`flutter analyze` 0問題、Flutter測試137／137；相對路徑Flutter Web build通過。
- 五個家庭接力題材在零資料試演與選集都可發現。3筆官方機會另有資格、報名角色、規則、查核／複查日、失效提示與官方簡章；未翻譯內容不能冒充越南語完成句。

2. Web行為回歸涵蓋試演四段實際音訊、兩個結果世界、五題材與App-owned storage前後比對；連同正式劇場、家庭接力與邀請握手，Chromium與Edge各6／6。
3. 瀏覽器回歸規格涵蓋零基礎「點場景圖選意思 → 點擊聽短句 → 正常／慢速／逐段聽 → 選配麥克風或直接用圖繼續 → 外婆同圖回話」，並驗證裝置聽寫不可用時不阻塞故事。
4. 瀏覽器回歸規格涵蓋三回合分支、兩次選擇一路保留到第三幕、可暫停換幕倒數、持續結果舞台、單一完成與家庭故事卡。
5. 瀏覽器回歸規格涵蓋五集故事選集、家長第一屏先顯示最新待回應故事、成人限定貼圖／家庭便條，以及過去故事與管理工具預設收合。
6. 瀏覽器回歸涵蓋完整成人邀請握手：管理者先展開收合工具區、建立一次性邀請包、受邀者本人設定六位數家人碼並產生簽章接受回覆、原家庭重新登入並再次展開工具後核准，最後由受邀者只用自己的家人碼進入；此案例包含在兩個瀏覽器各6／6的Web行為範圍。
7. Widget與瀏覽器回歸涵蓋五題材、3筆結構化官方機會及複查日期邊界；孩子完成成人確認、四關、三棒、家人圈與護照狀態轉移。未翻譯內容不能儲存為完成短句。

## 語音能力邊界

- 五個場景的中央聆聽按鈕固定播放完整句；測試逐一斷言它不會誤播答案詞塊或單獨句尾 `ạ`。
- 自然短語練習會把一個單詞的句尾接回前段；`cháu / về rồi / ạ` 實際提供 `cháu`、`về rồi ạ`，避免字典式孤立發音。
- 目前程式引用的 119 個離線 MP3 全部由同一個釘選 Piper 版本與 `vi_VN-vais1000-medium` 模型離線產生；App 執行時不依賴 Piper。產生器、模型、模型 SHA-256、授權、輸入文字、bytes 與逐檔 SHA-256 都有 manifest。
- 約30秒試演與正式劇場的每幕長輩首句都要由使用者明確點擊才播放；頁面載入不應產生音訊。使用者自行錄製的家人原音仍優先於內建合成示範。
- Round 15 的 Chromium 與 Edge 行為回歸均要求同一次明確點擊取得實際 MP3 HTTP 200，並由真正的 `HTMLAudioElement` 完整觀察 `play → playing → ended`、無 `error`、duration／currentTime 大於 0；只看到網路檔案存在或呼叫 `play()` 不算通過。
- 內建 MP3 無法播放時才誠實退回裝置 TTS；家庭自訂但尚未錄音的文字也使用裝置 TTS。TTS 會正規化 BCP-47 標籤並只選完全相符語音，不沿用中英文預設 voice。
- Web 與原生平台採不同語速校正；連點播放使用 latest-wins 並主動完成取消，避免舊句與新句重疊。
- `tool/bundled_audio_catalog.dart` 與 `piper_generation_manifest.json` 都以同一組 119 條路徑為基準；產生腳本、資產測試與同步檢查會攔截遺漏、空檔、非預期模型及 hash 衝突。
- Web 建置停用舊 Flutter service worker；入口會清除同一 App scope 的舊 registration 與 `flutter-*` cache，避免修正後仍被舊 `main.dart.js` 覆蓋。
- 劇場使用裝置／瀏覽器的 `speech_to_text`。transcript 只在當題已知選項中尋找關鍵詞；越南語比對可容忍聽寫漏掉聲調符號，但不據此宣稱聲調正確。
- 瀏覽器 confidence 為0代表未提供，不再被誤作低信心；任一供應商信心值都不作發音門檻。共同詞、多重命中、無聲音與權限失敗不會猜答案，會顯示慢速／逐段協助；孩子也可不開麥克風，直接用先前選好的圖讓外婆回話。
- 命中後畫面顯示實際「系統聽成：transcript」與「故事接到哪個意思」，不宣稱孩子說出完整標準句。
- 沒有音素、聲調、地方腔或發音正確率評量。
- 119 個 Piper 資產是開發期合成示範，不是家人原音或母語者驗收成果；自動化只能證明交付路徑、媒體事件、hash 與檔案可解析，不能證明越南語聲調、語氣或家庭稱謂正確。TTS 與聽寫由作業系統／瀏覽器提供，是否連網依平台設定；正式版仍須母語者實聽或家人重錄。

## 家庭圈與資料

- 首位成人必須明確建立家庭圈；不預置外婆、爸爸或孩子。
- 新增成人不能由舊的直接核准路徑加入。管理者只保存一次性邀請的 Ed25519 公鑰；私密 seed 只存在交給受邀者本人的邀請包。
- 受邀者本人設定六位數家人碼並簽出接受回覆；原家庭會驗證簽章、家庭 ID、成員 ID、期限與時間。過期、撤銷、竄改、錯家庭與重播都會拒絕。
- 核准後的受邀成人可只用自己的六位數家人碼進入回應故事，不需知道四位管理碼，也不能使用邀請、移除、備份、匯入或全部刪除等管理工具。
- 成人可切換與撤銷已核准家庭角色；限定貼圖可再次點擊撤回。
- 成人可按五集、三回合與所有分支節點保存家庭說法及本機原音；劇場會依實際走到的 prompt 優先播放對應家人原音，缺少錄音時清楚標示為裝置朗讀。
- 孩子可從選集首屏選擇「與家人分享、社團、午餐、上課、朋友關係」五個家庭接力題材；系統先讓孩子選第一棒，再顯示交回成人提示並要求 PIN。只有中文提示、尚無家人確認目標語時不能儲存；孩子完成看／聽／排／答四關後才建立三棒作品、家人圈項目與護照完成狀態。
- 選集提供 3 筆官方教材、課程與競賽資訊入口；這是可操作的外部資源導引，不代表已取得教育部合作、背書或採購。
- 成人家長頁第一屏只先整理最新待回應故事、反應、文字／錄音回覆與未讀狀態；過去故事、邀請、備份、成員與主劇情原音預設收合，不宣稱另一台裝置已即時送達。
- 家庭圈沒有陌生人搜尋、公開動態、私訊或排行榜。
- 文字資料包手動複製；錄音本體、本機 recording reference、個別家人碼 verifier／salt、待用／已用邀請與 retry state 均不匯出。
- 個別家人碼使用 PBKDF2-HMAC-SHA256 600,000 次、獨立 16-byte salt 與 32-byte verifier，固定時間比較；每位成人各自連續五次錯誤暫停 30 秒，重開後仍保留鎖定狀態。
- 舊版四位管理碼在成功驗證一次後會從快速 SHA-256 自動遷移到 PBKDF2 v2；管理碼與個別家人碼彼此分離。
- Widget 整合測試證明「全部刪除」會清除家庭圈、故事／學習資料與媒體，並回到同意閘門。

## 誠實性回歸

- 點卡顯示「你選了」，語音顯示實際 transcript。
- 劇場只建立「家庭故事卡」，沒有保存孩子原音就不稱聲音卡。
- 可達的舊短句流程在文字 fallback 時顯示「文字紀錄」，不再宣稱聲音已送進家人信箱。
- 品質閘門會阻擋供應商專屬名稱、假 AI 教練、發音正確、即時生成、聲音已放進與聲音卡加一等高風險文案。
- 約30秒試演必須維持無 store、recorder、speech recognizer 與完成回呼；第三幕只使用不可變的內建預演資料。若日後改變，必須重新做資料流與同意稽核。
