# 島語通 DaoTalk AI｜正式版嚴格驗收報告

驗收日期：2026-07-11（Asia/Taipei）  
驗收範圍：計畫書、正式網站、FastAPI 後端、多模態 adapter、安全規則、官方範本、合成文件、評測器、3 分 20 秒動畫與旁白。

## 結論

目前版本已經不是「一套 HTML 畫面」，而是一個能啟動、能收檔、能切換 mock／live、能固定輸出格式、能保留證據、能攔截高風險並可重跑評測的文件理解系統。正式網站、後端、合成案例、評測報告與影片使用相同的產品名稱與功能狀態；沒有把 mock、合成輸出或 adapter 測試包裝成 live AI 成績。

就初審作品完整度而言，現版具備明顯的「冠軍相」：題目聚焦、臺灣官方文件基礎清楚、操作畫面成熟、AI 架構可說明、安全界線可信、影片有記憶點。就冠軍所需的證據而言，仍有三個不能迴避的缺口：尚未以真金鑰跑多模態影像評測、尚未取得移工任務測試結果、目標語言尚未完成母語審閱。因此本次判定為：

**離線作品與送審素材通過；live AI 與第一手使用者證據為送件前最高優先補件。**

## 冠軍相已經成立的部分

### 問題與產品收斂

作品沒有再擴張到求職媒合、醫療、租屋或所有移工生活服務。整條故事只處理「收到中文職場文件後，如何找到重點、準備問題並安全求助」。評審能在首頁、計畫書與影片中看到同一條流程。

### AI 技術不再只是名詞

後端已有圖片、PDF 與文字輸入；影像採多模態 input，PDF 採文字與逐頁影像，回應由 Pydantic strict schema 驗證。模型負責版面與語意，本地程式負責風險補捉、提示注入拒絕、PII 回應遮罩及確定性計算。這比在計畫書只列 OCR、LLM、翻譯三個名詞更能回答技術成熟度。

### 臺灣情境有可追溯基礎

14 份官方範本覆蓋薪資、出勤、加班、請假、契約、職安、職災與移工通報。首波以中越薪資明細為主，是因為表格、雙語、金額、扣款、遮罩與重算都能客觀驗證，不是因為畫面比較好看。

### 失敗狀態是產品的一部分

沒有金鑰時，影像與 PDF 回覆低信心並明示未辨識；mock 畫面標示非 AI；高風險不下合法／違法結論；第三方保留政策沒有被 `store=false` 掩蓋。這些細節能提高評審對團隊可信度的判斷。

### 送審素材一致

新版計畫書使用正式介面截圖，影片第 7 幕說明多模態與 schema，第 10 幕同時呈現通過項目與 41.2% 的規則基線，旁白也明確說 live pending。作品不會出現「網站說已完成、計畫書說下一階段、影片又假裝即時 AI」的互相矛盾。

## 尚未通過冠軍門檻的部分

### 1. Live 多模態模型尚未實測

目前環境沒有 `OPENAI_API_KEY`。雖然 adapter 的 request shape、strict schema 與 `store=false` 都有測試，仍不能證明模型能正確讀取中越薪資單。至少要用同一模型跑：清晰 PNG、含背景 PNG、PDF、模糊小字、旋轉、金額差異、提示注入與低信心案例，保存原始 JSON、模型版次、延遲與成本。

### 2. 還沒有移工完成同版任務

官方人口數只能證明服務範圍，不能證明產品好用。最少要有 5 位目標使用者完成同一版本，記錄是否能說出文件用途、找出 3,500 元扣款、說出下一步、找到 1955，以及完成時間與提示次數。沒有這些證據，市場價值仍是最大失分區。

### 3. 目標語言尚未完成正式審閱

現有越南語、印尼語與英語可驗證資料契約與介面，但在熟悉臺灣職場用語的母語者審閱前，不能標示正式支援。至少保留審閱者代碼、日期、原句、修改後句子與修改原因。

### 4. 敏感資訊仍需在送出前自動處理

後端已能辨識 PII 類型並遮罩回應，但影像在送往外部模型前的本地自動遮罩尚未完成。目前只能要求使用者先遮住。正式公開服務前，應補本地或受控後端的欄位偵測、預覽確認與不可逆遮罩。

### 5. 尚未完成正式營運部署

競賽 demo 已可同源啟動，但公開網址仍需 HTTPS、secret manager、身分或濫用控制、rate limit、監控告警、錯誤回報與刪除窗口。Safari、Firefox、實際螢幕閱讀器與完整 WCAG 對比稽核也尚未完成。

## 送件前優先順序

1. **先跑 live 模型。** 只用完全合成資料；產出影像／PDF exact match、風險召回、證據引用、延遲與成本報告。
2. **完成至少 5 位移工任務測試。** 同版、同文件、同指標；不要只收滿意度。
3. **完成越南語審閱。** 首波只承諾一個真正測過的目標語言，比同時寫四語更可信。
4. **部署 HTTPS 固定網址。** 用無痕視窗、Android、iPhone、兩種瀏覽器和慢速網路驗收；固定 QR code。
5. **錄製 3:20 影片。** 使用 `video/?record=1`，依逐幕稿配音；補入 live 與真人數據前維持 pending 標籤。
6. **最後同步四份材料。** 網站、影片、計畫書、評測報告的模型、數字、語言與完成狀態必須完全一致。

## 最終 Go／No-Go

- **若初審今天截止：Go。** 可提交目前計畫書、正式網站與影片畫面；所有未驗證能力保持原標籤，不造數。
- **若仍有一至兩週：有條件 Go。** 不再擴功能，集中完成 live 評測、5 位移工任務與越南語審閱；這三項最能提高勝率。
- **No-Go 條件：** 把 mock 41.2% 改寫成 AI 準確率、把目標樣本數寫成已訪談、把未審閱越南語寫成正式支援，或在影片中移除 synthetic／pending 標籤。任何一項都會直接傷害可信度。
