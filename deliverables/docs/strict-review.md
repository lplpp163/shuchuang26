## 結論先說

「傳家話」已從「看圖選答案的語言題庫」重做成「孩子與長輩輪流演出的家庭對話劇場」。孩子先看懂中文意思，再聽短句、看羅馬拼音並開口；語音不可用時有誠實的小卡替代，不會假裝孩子說過。回答會改變角色下一句、整個故事舞台與後續分支，三幕後只完成一次，再把故事卡留到私密家庭圈讓成人核准的家庭角色稍後接續。五個日常題材也已形成可完成的三棒家庭接力：孩子選第一棒、家人確認真正說法，孩子走完看／聽／排／答四關後才揭曉作品並替故事護照蓋章。

## 目前實際可玩的核心

- 五個 60–90 秒、三幕式生活故事：放學回家、早晨起床、一起準備晚餐、陽台澆花、睡前故事。
- 選集首屏另有五個可操作的家庭接力題材：與家人分享、社團、午餐、上課、朋友關係。孩子選第一棒後把裝置交回家人確認真正說法，再完成看／聽／排／答四關；只有完整走完才揭曉三棒作品、收進家人圈並替故事護照蓋章。
- 提供 3 筆官方教材、課程與說故事競賽資訊入口，讓家庭與教師能由 App 連到實際教育資源；資訊入口不等同已取得教育部合作或採購。
- 每幕由長輩先說；第一句必須由孩子明確點擊播放，再選想表達的中文意思，逐步看見目標語、羅馬拼音、中文與播放控制。
- 支援裝置／瀏覽器聽寫；只比對本題已審核關鍵詞，模糊、低信心或同時碰到兩個意圖時會自然請孩子確認。
- 沒有語音權限或不想開口時，可用明確標示的意思小卡繼續；點卡顯示「你選了」，只有真的收到聽寫結果才顯示「系統聽成」。
- 回答後角色會回一句，舞台角色、路徑、顏色與場景物件一起改變；第一、第二個選擇都會影響第三幕節點。
- 完成後建立一張包含完整選擇路徑的家庭故事卡，不重複慶祝或產生兩次完成。
- 成人核准的家庭角色可留限定貼圖、文字、錄音與家庭記憶；孩子不必等待家人回覆就能完成當下故事。
- 新增成人必須經過「管理者建立一次性邀請包 → 受邀者本人設定六位數家人碼並交回簽章回覆 → 原家庭核准」；待核准角色不會出現在孩子畫面。
- 家人能替內建劇場錄「我們家版本」並在主故事中優先播放；裝置朗讀與家人原音必須使用不同標示。
- 家人也能以幾行話確認接力短句；目前只對五種已知場景提供本機模板與作品內建圖片，不假裝即時生成或任意翻譯。

## 技術誠實邊界

- 目前沒有假稱遠端生成、即時生圖或任意語言翻譯。故事是可檢查的本機狀態機。
- 場景圖是開發期間製作的生成式合成插圖，不是實拍或使用者證言；10 次生成到 11 張成品的原始輸出、完整 prompt、匿名化紀錄、引用／後製對照與逐檔雜湊已封存，公開 App 只帶匿名 manifest。這是技術來源與完整性證據，不等於送件或商用權利已通過；人工權利聲明仍待有權限者本人簽署。
- 聽寫文字可能由裝置或瀏覽器平台處理；本作只用回傳文字找關鍵詞，不以錄音長度、音量或一般聽寫宣稱能評估音素、聲調或家庭口音。
- 越南語腳本、羅馬拼音與隨包 Piper 合成示範尚未完成母語者逐句審閱，不能宣稱是標準答案或家庭自然語氣。
- 家庭圈具備手動交換的一次性成人邀請、本人六位數家人碼、簽章接受回覆、原家庭核准、撤銷與個別驗證；它不會自動把邀請送到另一台裝置，也沒有公開動態、陌生人搜尋、私訊或排行榜。
- JSON 資料包可匯出／匯入文字、成員與互動狀態，但會清除本機錄音 reference，且不包含音檔本體。
- 四位管理碼與每位受邀成人的六位數家人碼分開；個別家人碼使用 PBKDF2-HMAC-SHA256 600,000 次與獨立 salt，連續五次錯誤暫停 30 秒。這仍是原型級本機驗證，不等同正式帳號、帳號復原或端對端加密。
- 可攜式家庭備份會排除家人碼 verifier、salt、待用／已用邀請與錯誤重試狀態；個別憑證不會跟著文字資料包搬移。

## 與成熟產品相比仍缺什麼

官方產品 benchmark 原本指出「每句慢速／逐段聽」與「非競爭家庭共同任務」兩個缺口，Round 6 已完成並有測試與畫面證據。下一階段真正拉開差距的不是再堆卡片，而是以下四件事：

1. 供應商中立的即時對話引擎：孩子能自由回答，角色能請他重說、放慢或自然接話；沒有 API 時仍回退現有安全分支。
2. 更像故事的錯誤後果：答偏時角色拿錯道具、困惑或重新確認，而不是紅叉與扣分。
3. 成人預覽核准的自建小劇場：輸入「誰說、發生什麼、想留下哪句」，形成三幕草稿而非只存一張短句卡。
4. 伺服器帳號、安全跨裝置同步、帳號復原與推播；手動邀請握手不能冒充遠端送達或雲端社群。

## 送件前仍需真人完成

1. 請至少一位越南語母語成人逐句檢查五集文字、稱謂、語氣與羅馬拼音；保留修訂紀錄。
2. 以 3–5 位幾乎不會越南語的孩子實測：第一次是否不用解說、是否知道何時輪到自己、是否願意重玩，以及哪裡會把平板交回成人。
3. 用 Android、iPhone 與兩種桌面瀏覽器驗證真實麥克風權限、兒童音高／口音聽寫、家人原音播放、重新整理後保存與全部資料刪除。
4. 若要宣稱產品級雲端私密社群，必須補伺服器帳號、安全跨裝置送達／同步、撤權傳播、帳號復原與威脅模型；目前只能稱具本人接受與個別驗證的本機手動家庭圈。
5. 部署公開 HTTPS 並以無痕視窗重走冷啟動；現有本機靜態伺服器證據不能取代公開網址。

## 最終判決

核心已不是「聽一段音、點一張圖」；冷啟三幕先讓評審看見一句話如何從孩子帶回、由家人傳下，再被孩子接住，正式流程才取得同意、建立家庭並保存作品。產品也能展示孩子與長輩輪流說、故事因孩子選擇改變、使用者自行錄製的家人原音回到角色，以及成人核准的家庭角色稍後接續但不阻塞孩子。五個日常題材已能走完「孩子第一棒 → 家人真正說法 → 孩子四關 → 三棒揭曉 → 家人圈／故事護照」，3 筆官方教材／課程／競賽資訊也各有明確區隔的本機延伸。`flutter analyze` 0 問題、134／134 項測試與相對路徑 Web build通過；Chromium與Edge的App回歸各7／7、同步網站完整套件各10／10，Edge有聲媒體2／2。正式計畫書、影片、公開provenance manifest與三個Android分架構候選包均須以最終同步清單的尺寸與雜湊為準。嚴格分數維持74／100、市場16／30，正式送審仍是條件式NO-GO，直到公開HTTPS、行政附件、真人／母語／買方證據與人工圖像權利聲明完成。

仍不能宣稱的是：可靠的兒童發音評估、已經母語審閱、產品級跨裝置雲端社群、任意文字即時生成完整圖片與聲音、或真人家庭長期留存。這些外部證據會決定作品能否從「完整原型」進一步成為令人為之一亮、且值得長期使用的文化傳承產品。
